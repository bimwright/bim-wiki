# bim-wiki Roadmap ‚Äî Design v1

Written: 2026-04-24. Supersedes: none (first roadmap).

## 1. Context

bim-wiki v0.1 published 2026-04-19 with 56 markdown pages covering ISO 19650 Parts 1‚Äì6 and the Vietnamese regulatory landscape (Qƒê 347/348/1057/2500, Lu·∫≠t 60/2024, Nƒê 111/2024, Nƒê 175/2024, TT 09/2024, TT 24/2025, etc.). Two-week post-launch traffic: 225 views / 103 unique, 78 clones / 50 unique, 1 star, 0 issues, 0 PRs. Referrers: github.com (16), chat.zalo.me (3). Cooling rapidly after launch spike ‚Äî single broadcast, no tail.

Structure is flat markdown at repo root. Index (`index.md`) is agent-maintained. Evidence tags in place: `[ISO-CORE:verified]`, `[ISO-CORE:structure]`, `[VN-BXD:verified]`, `[COMMENTARY]`, `[INFERRED]`. Frontmatter minimal (`title`, `type`, `updated`). No `.github/`, no CI, no contribution scaffold. READMEs in EN / VI / zh-CN; content is EN-only with Vietnamese technical terms preserved.

This roadmap governs the next expansion phase.

## 2. Goals

Primary (ranked):

- **G1 ‚Äî Depth / coverage.** Fill content gaps: openBIM schema backbone (IFC mentioned 14 times, BCF 1, COBie 2, IDM 1, MVD/bSDD 0 across existing pages ‚Äî referenced but never defined), VN regulatory deepening with worked examples, QCVN/TCVN bridge pages. Add `[PRACTICE]` tier.
- **G2 ‚Äî Agent-readiness (minimum viable).** Make pages LLM-consumable without committing to full embeddings/MCP infrastructure yet. Stable schema, `summary_short` field, `llms.txt`, clean wiki-link graph.
- **G3 ‚Äî Open layer.** Connect BIM information-management core to adjacent knowledge: openBIM ecosystem (IFC/BCF/COBie/IDM/MVD/bSDD/IFD), VN technical codes (QCVN/TCVN structural/MEP/fire), practitioner experience (anonymized BEP/EIR worked examples), international comparatives (UK/Singapore/China/EU).

## 3. Non-goals

- Authority via external citation. Primary sources (ISO, Nƒê, Qƒê, TT) will be cited directly ‚Äî we are a map, not a substitute. Do not optimize for "be quoted in BEPs."
- Full AI grounding infrastructure (embeddings, vector DB, MCP server, evaluation harness). Deferred until the wiki has contributors and sustained traffic to justify the infra cost.
- Rewriting QCVN/TCVN. Scope is bridge pages only ("when BEP needs to spec MEP, reference which QCVN/TCVN, map to which IFC fields").
- Tool tutorials. Tooling pages only cover behavior that affects ISO 19650 concepts (e.g., Revit worksets ‚Üî federation). No "how to draw a wall in Revit."
- Fixed delivery cadence. Theme-sprint mode; sprints ship when done.

## 4. Success signals

- Content: from 56 ‚Üí ~120 pages within the first 3 theme sprints. No orphans (every page ‚â•1 inbound + ‚â•1 outbound wiki-link).
- Agent-ready: `llms.txt` + `summary_short` on every page, schema lint passes in CI, a sample LLM retrieval query against the wiki returns the correct page as top result for ‚â•80% of a 20-question test set (questions seeded from `summary_short` sentences).
- Open layer: openBIM core (7 pages) + glossary + disambiguation landed; ‚â•1 VN worked example; ‚â•1 international comparative expanded beyond stub.
- Health: page updated-stamps stay fresh (no page >12 months untouched for `stable` status); `living` pages track regulation changes within 2 weeks of publication.

Not measured: stars, forks, external citations. These are downstream.

## 5. Principles

1. **Portfolio, not timeline.** Sprints are independent units picked by theme, not scheduled by week. A sprint may last 3 days or 3 weeks.
2. **2:1 content-to-infra ratio.** Every two content sprints are followed by one infra sprint. Prevents drift toward unbounded content at the cost of structural decay.
3. **One sprint at a time.** No parallel sprints. Context switching breaks DoD discipline.
4. **Flat structure preserved.** No folder nesting for content pages. Wiki-links are flat (`[[slug]]`) and must remain flat. Grouping is metadata (`type`, `tags`), not filesystem.
5. **Evidence tag discipline.** Every page carries ‚â•1 page-level evidence tag in frontmatter. Inline tags (at sentence or paragraph level) are required for any claim that would surprise a careful reader or that a practitioner might act on. Existing taxonomy extended with `[PRACTICE]` and `[TOOLING]`.
6. **Self-contained sprints.** Each sprint must define entry, scope, and exit before starting. If any of the three can't be stated in one sentence, the sprint isn't ready.

## 6. Sprint anatomy

```
Entry
  - Theme statement (one sentence: what and why now)
  - Source list (PDFs, standards, URLs, commit hashes for living sources)
  - Pages in scope: CREATE / EDIT / TOUCH-BACKLINK lists

Scope
  - Content deliverable (N new/updated pages)
  - Schema deliverable (new fields/tags, if any)
  - Link deliverable (which existing pages gain inbound links)

Exit ‚Äî Definition of Done (7 points)
  1. Every new page has complete frontmatter (title, type, status, updated, summary_short, tags)
  2. Every page has ‚â•1 page-level evidence tag; non-obvious claims carry inline tags
  3. index.md updated (slug + summary_short ‚â§120 chars for index line)
  4. No orphan pages (‚â•1 inbound + ‚â•1 outbound wiki-link)
  5. Language parity noted (page marked lang: [en], [en, vi], or [en, vi, zh-CN])
  6. Single commit with subject format: `feat(sprint-N-<slug>): <what>` + body notes source commit/PDF
  7. Page and index `updated:` stamps bumped
```

## 7. Schema v1

Locked in Sprint 0. Every page must conform.

```yaml
---
title: string                    # required
type: entity|concept|comparison|summary|meta|overview|synthesis|path|disambiguation|cheatsheet|glossary|roadmap
status: draft|stable|living|superseded  # required
updated: YYYY-MM-DD              # required
summary_short: string ‚â§160 chars # required ‚Äî LLM retrieval + social share
tags: [ISO-CORE:verified, ISO-CORE:structure, VN-BXD:verified, COMMENTARY, INFERRED, PRACTICE, TOOLING]
applies_to:                      # optional ‚Äî for VN-specific regulatory pages
  cap: [I, II, III, IV]
  nhom: [A, B, C]
  nguon_von: [cong, tu-nhan, ppp]
supersedes: [slug]               # optional
superseded_by: slug              # optional
prerequisites: [slug]            # optional ‚Äî recommended reading order
lang: [en, vi, zh-CN]            # optional ‚Äî default [en]
---
```

Rationale:

- `status` with `living` value: pages tracking regulation or standard revisions (e.g., `iso-19650-revision-watch`, future `vn-bim-regulation-watch`).
- `summary_short`: LLM retrieval needs a dedicated 1-sentence abstract, ‚â§160 chars, distinct from the 120-char TL;DR in `index.md`. Also doubles as social-share text.
- `supersedes` / `superseded_by`: VN policy churns. Qƒê 2500 ‚Üí Qƒê 258, Qƒê 1057 ‚Üí Qƒê 348. Track these chains to avoid stale guidance.
- `applies_to`: For pages whose applicability depends on project class, group, or funding source. Matches the Nƒê 175/2024 and Qƒê 258 axes.
- `prerequisites`: Seeds Tier-2 concept-dependency feature.
- `lang`: Current content is EN-only. Field records this explicitly and enables future parity audits.
- `type` enum values `path` and `cheatsheet` are reserved: no pages of these types ship in v1. `path` will be used when reading paths (F1) land after the ~80-page threshold. `cheatsheet` is reserved for a potential future F4 adoption but is not planned in the current roadmap.

## 9. Creative features adopted

Four features from broader wiki ecosystems integrated into schema and early sprints:

- **F1 ‚Äî Reading paths** (inspiration: Arch Wiki, MDN Learn). Curated linear sequences linking existing content. Pages `path-<role>.md` with `type: path`. No new domain content. Deferred until ~80 pages.
- **F2 ‚Äî Trilingual glossary** (inspiration: Wikipedia terminology tables). Single page `glossary.md` with EN / VI / zh-CN columns. Seeded in Sprint 1 with openBIM terms. Every sprint appends.
- **F3 ‚Äî Disambiguation pages** (inspiration: Wikipedia). `<term>-disambiguation.md` for terms with multiple meanings (LOD, IFC, BIM). Type `disambiguation`. First instance: `ifc-disambiguation.md` in Sprint 1.
- **F5 ‚Äî `summary_short` frontmatter** (inspiration: Anthropic `llms.txt` spec). Locked in schema v1 (Sprint 0).

Not adopted in v1: cheat sheets (F4, risk of content duplication with concept pages), concept-dependency tree (defer), version matrix (defer until ISO 19650 DIS 2026 nears), role-based landing pages (overlap with F1), talk pages, quiz, FAQ, GitHub Pages, OpenGraph cards, semantic "see also." All require higher content volume or community signal than currently exists.

## 10. Sprint 0 ‚Äî Schema Hardening & Agent-Ready Baseline

**Entry**

Audit the 56 existing pages' frontmatter. List fields in use, fields missing, inconsistency. Reference: current frontmatter has `title`, `updated`, partial `type`. Missing: `status`, `summary_short`, versioning, `lang`.

**Scope**

1. `schema.md` at repo root ‚Äî single-page spec defining frontmatter fields, value enums, and tag taxonomy (see ¬ß7 above).
2. Migration of all 56 pages to schema v1:
   - Write `summary_short` ‚â§160 chars for each page (distinct from the 120-char index line).
   - Set `status`: default `stable`; `living` for `iso-19650-revision-watch`; `superseded` notation for pages mapping obsolete Qƒê (1057 ‚Üí 348, 2500 ‚Üí 258).
   - Populate `supersedes` / `superseded_by` where applicable.
   - Mark `lang: [en]` on all current pages.
3. `scripts/lint_frontmatter.py` ‚Äî validates required fields, enum values, tag membership, supersedes chain integrity. Emits JSON + human-readable output.
4. GitHub Action: `.github/workflows/lint.yml` ‚Äî runs the linter on every PR and push to master.
5. Tag taxonomy expansion: document `[PRACTICE]` and `[TOOLING]` semantics in `schema.md`, reserved for later sprints.

**Exit ‚Äî DoD**

1. 56/56 pages pass `lint_frontmatter.py`.
2. Every page has ‚â•1 page-level evidence tag in frontmatter (retrofit where missing).
3. `index.md` regenerated; TL;DRs derived from `summary_short`.
4. Wiki-link graph unchanged; no broken `[[slug]]` references.
5. All pages marked `lang: [en]`.
6. Commit: `feat(sprint-0-schema): harden frontmatter + lint + migration`.
7. `schema.md` and `index.md` `updated:` stamps set to sprint-end date.

**Estimated effort:** 2‚Äì3 days with agent assistance on migration.

## 13. Risks & mitigations

- **Schema churn.** Locking schema v1 then discovering it's insufficient for C3/C4/C5. *Mitigation:* schema is additive; new optional fields allowed without migration. Required fields change only via a v2 with explicit migration sprint.
- **Scope creep in C3 (QCVN/TCVN).** Temptation to rewrite codes. *Mitigation:* bridge-page rule explicit in ¬ß3 non-goals; reviewer enforces at exit.
- **Cross-language debt.** Adding EN content faster than VI/zh-CN can keep up. *Mitigation:* `lang` field makes debt visible; I4 sprint scheduled before content passes 100 pages.
- **Agent-ingest drift.** If the ingest agent and the wiki diverge (e.g., index.md falls out of sync with disk), downstream readers break. *Mitigation:* I1 lint script checks drift; runs in CI.
- **Abandonment after launch afterglow.** Traffic already cooling. *Mitigation:* ship Sprint 0 fast (2‚Äì3 days); visible progress sustains author motivation.
